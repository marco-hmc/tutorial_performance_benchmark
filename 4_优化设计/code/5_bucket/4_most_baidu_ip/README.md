# 题目：从海量日志中提取某日访问百度次数最多的 IP（示例实现）

问题描述：给定海量日志文件，提取某日访问百度（例如请求中包含 "www.baidu.com"）的次数最多的 IP 地址。假设日志量很大，内存不足以一次性统计全部数据。

方案（按照笔记）：

- 第一步：预处理 — 从日志中筛选出当日访问百度的条目，并提取出 IP 地址。
- 第二步：分桶 — 对 IP 做 `hash(IP) % 1000`，将记录写入 1000 个小文件，保证同一 IP 都落到同一桶。
- 第三步：局部统计 — 对每个小文件使用 `unordered_map` 统计 IP 出现次数，记录每桶的局部最大 IP 及其次数。
- 第四步：汇总 — 在 1000 个局部结果中找到全局最大值。

本示例提供一个小规模可运行的实现（使用 10 个桶以便在本地演示），包含 memory 模式（直接统计）和 bucket 模式（分桶统计再汇总）。

如何运行：

```bash
cd 5_数据设计/code/4_most_baidu_ip
./run_test.sh
```
