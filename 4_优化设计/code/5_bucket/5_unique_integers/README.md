# 题目：在 2.5 亿个整数中找出不重复的整数（示例实现）

问题描述（摘自笔记）：给定大量整数（例如 2.5 亿个），内存无法一次性容纳所有数据，要求找出仅出现一次的整数集合。

方案（笔记中给出两种）：

- 方案 1（2-Bitmap）: 对每个可能的整数使用 2 bit 表示出现状态：00=未出现，01=出现一次，10=出现多次。扫描数据流一次即可得到只出现一次的数。此方法对域有界（例如 32 位整数）且内存允许时非常高效。
- 方案 2（分治/分桶）: 将数据按 `hash(x)%B` 分到 B 个小文件，保证相同数字落在同一桶。对每个桶在内存中统计出现次数并输出桶内出现一次的数字，最后汇总各桶结果。

本目录提供小规模可运行的示例：

- `unique_integers.cpp`：实现了两种示例方法
  - `bitmap_demo`：基于 2-bit 位图（按输入中最大值分配位图，示例对较小的值域有效）
  - `partition_demo`：按 `hash(x)%B` 分桶，示例使用 B=4
- `sample_numbers.txt`：小规模示例输入
- `run_test.sh`：编译并执行两种方法

注意：真实处理 2.5 亿或更大的数据时，请按实际环境选择 B 并使用稳定的临时文件目录；位图方法在处理 32-bit 全域时需要约 1GB 内存（2-bit × 2^32）。示例代码里会在超出阈值时提示并建议使用分桶方法。
