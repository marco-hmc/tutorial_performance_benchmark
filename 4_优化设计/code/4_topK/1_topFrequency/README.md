# 题目：找出 1G 文件中频数最高的 100 个词（示例实现）

问题描述（摘自笔记）：有一个 1G 大小的文件，每行一个词，词大小不超过 16 字节，内存限制 1M。要求找出出现频数最高的 100 个词。

实现思路（按照笔记给定方案）：

- 第一阶段：划分（外部分桶）

  - 使用 `hash(word) % 5000` 将原始大文件分为 5000 个小文件。此处示例使用较小的 bucket 数以方便本地测试。
  - 若某个 bucket 文件仍然超过内存限制，可对该 bucket 继续按 hash 细分（代码中提供二级分桶逻辑的演示）。

- 第二阶段：局部统计

  - 对每个小文件使用 `unordered_map<string,int>` 统计词频（单个小文件大小保证可在内存中处理），然后用大小为 100 的小根堆选出本桶 Top-100，写入候选文件。

- 第三阶段：全局归并
  - 将所有桶的候选 Top-100 合并到一个全局大小为 100 的小根堆，得到最终 Top-100。

注意：本仓库中的示例数据规模较小，仅用于验证算法正确性与流程。真实 1G 数据需要在生产环境中根据内存与磁盘进行参数调整（例如 bucket 数、临时目录位置等）。

如何运行：

进入目录并执行：

```bash
./run_test.sh
```

该脚本会编译 `top100_words.cpp`，并用 `sample.txt` 做一个小规模演示，分别运行 memory（直接统计）与 bucket（分桶后统计再合并）两种模式。
