# 题目 1：找出 a、b 两个文件共同的 URL

完整题目说明：

给定两个包含大量 URL 的文件 `a.txt` 和 `b.txt`，每行一个 URL。假设每个文件可能非常大（例如数十亿条），内存受限（例如 4GB）。请设计并实现一种方法，找出两个文件中的交集（共同出现的 URL）。 **分析**：每个文件大小估计为 50 亿 ×64 字节 = 320G，远大于 4G 内存限制。

要求与说明：

- 提供两种可运行的模式：
  1. 内存模式（Memory mode）：将较小的文件整体读入内存的哈希集合，然后扫描另一个文件查找交集（仅用于小文件或测试）。
  2. 分桶模式（Bucket mode）：对两个文件按 hash(url)%M 分到 M 个桶（临时文件），然后逐桶在内存中处理（适用于海量数据）。
  3. **方案 1（分治+哈希）**：
  - 遍历文件 a，对每个 url 求哈希值，根据哈希值将 url 分别存储到 1000 个小文件中，每个小文件大小约为 320M
  - 遍历文件 b，采取相同方式将 url 存储到 1000 个小文件中
  - 所有可能相同的 url 都在对应的小文件中，求每对小文件中相同的 url
  - **处理不均衡**：若哈希分的不均衡导致有文件较大，应继续对较大文件进行哈希再分
  4. **方案 2（Bloom Filter）**：
  - 如果允许一定错误率，使用 Bloom filter
  - 4G 内存大概可表示 340 亿 bit
  - 将一个文件的 url 用 Bloom filter 映射，另一个文件逐个检查

目录内容：

- `find_common_urls.cpp`：实现（支持两种模式，命令行参数可选）
- `a.txt`, `b.txt`：小规模示例输入
- `run_test.sh`：编译并运行示例（展示两种模式）

示例运行： ./run_test.sh
